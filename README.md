# Тестирование CallRoute ML

## Структура тестов

```
tests/
├── conftest.py          # Фикстуры, настройка тестовой БД
├── test_auth.py         # Аутентификация и пользователи
├── test_billing.py      # Биллинг и транзакции
├── test_predict.py      # Предикты (ML-задачи)
└── test_admin.py        # Административный роутер
```

## Что покрывают тесты

### `test_auth.py`
- Регистрация: успех, дубль email, слабый пароль, начисление бонуса
- Логин: успех, неверный пароль, несуществующий пользователь
- JWT: валидный/невалидный токен, содержимое payload
- Профиль `/me` и баланс `/me/balance`
- Реферальная программа: создание кода, бонус рефереру, невалидный код
- Роли: user не может зайти в /admin
  
### `test_billing.py`
- Баланс: структура ответа, кредиты после регистрации
- Пополнение `/billing/topup`: успех, несколько пополнений, валидация (0, >10000, отрицательное)
- Транзакции: фильтр по типу, сортировка по дате, изоляция между пользователями
- Тарифы `/billing/tiers`: наличие fast/smart
- Лояльность `/billing/loyalty`: уровень bronze по умолчанию, прогресс до следующего уровня

### `test_predict.py`
- Создание задачи: успех, оплата из бонуса, нехватка кредитов (402), короткий текст (422), несуществующий тариф (400)
- Идемпотентность: повторный запрос с тем же ключом возвращает ту же задачу
- Тарифы fast и smart
- Получение задачи по ID: pending-статус, 404 для чужой задачи
- Список задач: пагинация, фильтр по статусу, изоляция между пользователями

### `test_admin.py`
- Статистика `/admin/stats`: доступна только админу, все поля числовые и >= 0
- Список пользователей `/admin/users`: структура, нет хеша пароля, новый юзер появляется
- Изменение тарифа `/admin/tiers/{name}`: успех, несуществующий тариф (404)
- Изменение уровня лояльности `/admin/loyalty/{name}`: успех, несуществующий (404)
- Все эндпоинты возвращают 403 для обычного пользователя и 401 без токена

## Подготовка тестовой БД
```powershell
# Убедитесь что запущены postgres и redis
docker-compose up -d postgres redis

# Создать тестовую БД
docker exec -it callroute_ml-postgres-1 psql -U callroute -d postgres -c "CREATE DATABASE callroute_test;"
```

## Примечания
- Тесты используют отдельную БД `callroute_test` — продовая БД не затрагивается
- Celery-воркер для тестов не нужен (замокан через `unittest.mock`)
- Каждый тест создаёт уникального пользователя — тесты не зависят друг от друга
